Inline Meeting Title in Notes Panel

Design spec  ·  2026-05-18  ·  Adamant v0.5.x

Overview

Problem

The Notes panel header always shows the static label "My Notes". Users have no way to see — or quickly edit — the meeting's title from the Notes panel. Renaming currently requires going through the AI Cleanup panel or sidebar, which is indirect.

Goal

Replace My Notes with the meeting's actual title, rendered as an inline-editable heading. Hovering reveals a pencil affordance; clicking converts it to a focused input. Saving persists to the database and syncs the sidebar — exactly as the existing api_save_meeting_title flow already does.

Current State

  • NotesPanel.tsx:599 — <h2>My Notes</h2>, static, no title prop
  • useMeetingData.ts — meetingTitle, handleTitleChange, handleSaveMeetingTitle already exist and work, but are not wired to the Notes panel
  • Backend — api_save_meeting_title Tauri command is complete; sidebar sync logic is in handleSaveMeetingTitle
  • page-content.tsx — renders NotesPanel but passes no title props; meetingData.meetingTitle and handlers are available right there

The backend, state management, and sidebar sync are all done. This is a purely frontend wiring + UI task.

Interaction Design

Keyboard & mouse behaviour

  • Click title → focus input, select all text
  • Enter → trim, save if changed, return to heading view
  • Escape → discard edits, return to heading view
  • Blur (click outside) → same as Enter (save if changed)
  • Empty input on save → revert to previous title (no empty titles)
  • Draft note (id==="new") → title shows Untitled Note in muted color, click-to-edit disabled until meeting is persisted

Save behaviour

  • Calls handleSaveMeetingTitle() from useMeetingData — already handles API call + sidebar sync + current meeting context update
  • No new API or state needed; just wire existing hooks to new props
  • Show a brief toast on save error (already handled by handleSaveMeetingTitle's error path)

Approaches Considered

A · Inline in NotesPanel Recommended

Add meetingTitle, onTitleChange, onTitleSave props to NotesPanel. Build the editable heading inline — a small controlled input toggled by isEditingTitle state local to NotesPanel.

✓ Self-contained. No new component file. Minimal prop surface. Follows the pattern already used in the panel (e.g. save status).

✗ NotesPanel gets a few more props.

B · Shared EditableTitle component

Extract a generic <EditableTitle> component, reuse it across Notes, Transcript, and Summary panels.

✓ Reusable if other panels ever need editable titles.

✗ Only one panel needs it now. Premature abstraction. Adds a new file for no current gain.

C · Title in a shared panel header component

Refactor all three panels to share a PaneHeader component that supports an optional editable title slot.

✗ Large refactor of working code. Much more risk for no user-facing gain. Out of scope.

Recommendation: Approach A. It's the smallest change that fully delivers the feature. The codebase explicitly avoids premature abstractions (per CLAUDE.md).

End Result

When a user opens any meeting, the Notes panel header shows the meeting's real name instead of "My Notes." Hovering over the title reveals a pencil icon. Clicking it replaces the heading with a text input pre-filled with the current title. The user types a new name and presses Enter (or clicks away) — the title saves instantly, updates in the sidebar, and the heading snaps back. Pressing Escape cancels without saving. For brand-new unsaved notes, the title displays "Untitled Note" in grey and is not editable until the note is first saved.
