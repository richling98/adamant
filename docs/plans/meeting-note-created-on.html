Meeting Note “Created On” Plan

Add the note creation date next to “My Notes” on the meeting notes page.

Overall progress: 100% Status: complete and manually reviewed Created: May 9, 2026

Answer: Is This Data Already Stored?

Yes. Creation date data already exists.

  • The meetings table has created_at for every meeting note, and the meeting details page already receives it as meeting.created_at.
  • The meeting_notes table also has created_at, but api_get_note currently returns only note content and updated_at.

Recommendation: use meeting.created_at for this feature. It is already available for all prior meeting notes, so retroactive support needs no data migration.

TLDR

Pass the already-loaded meeting creation timestamp into NotesPanel, format it for display, and render it next to the “My Notes” heading.

Mock Result

My Notes created on: May 9, 2026

Desired End Result

  • Every existing meeting note shows its original created date next to My Notes.
  • Every new meeting note shows the correct created date as soon as the page has a real persisted meeting record.
  • No database migration is required because prior meeting records already have creation timestamps.

Critical Decisions

Use meetings.created_at, not meeting_notes.created_at. The visible page is a meeting note page, and the meeting row exists before note content is saved. This gives consistent retroactive coverage.

Keep backend unchanged. The needed timestamp already flows through MeetingDetailsResponse into PageContent.

Format in the frontend. Use local date formatting, matching the app’s existing By Date/sidebar behavior.
