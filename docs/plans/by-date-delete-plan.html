By Date Delete Feature Plan

Add date-level selection and deletion controls to the existing By Date sidebar groups.

Overall progress: 100% Current status: complete and manually tested Last updated: May 9, 2026 9:16 PM Primary file: frontend/src/components/Sidebar/index.tsx

TLDR

The feature should reuse the current meeting bulk-delete system. A selected date will select every meeting ID in that date group, so the existing bottom delete bar and delete API can do the actual work.

Key Changes

  • Hovering a date row in By Date reveals a checkbox.
  • Checking a date selects every meeting under that date.
  • Users can check multiple dates and bulk-delete all selected meetings.
  • A date row gets a trash action for deleting that one whole date group.
  • Individual meeting rows inside By Date get the usual checkbox and delete controls.

Desired End Result

  • Delete all meetings from a date like 5/9/2026 in one action.
  • Select several dates, then delete all meetings across those dates.
  • Keep expand/collapse and click-to-open behavior working normally.

Critical Decisions

Reuse selectedMeetingIds as the source of truth. Selecting a date adds or removes all meeting IDs for that date, avoiding a second selection system.

Delete by meeting IDs, not by date string. The date label is just UI grouping; the real records are meetings.

Keep backend unchanged initially. Existing api_delete_meeting already deletes related transcript and summary data safely.

Add separate date-delete confirmation state. Date deletion needs text like “Delete all N meetings from DATE?” rather than the current single-meeting prompt.

Tasks

Step 1: Refactor By Date Group Data

  • Keep the existing groupsByDate logic.
  • For each date group, derive dateMeetingIds.
  • Derive all-selected, none-selected, and optional partial-selected state.

Step 2: Add Date Header Controls

  • Make each date header row a hover group.
  • Add a checkbox icon hidden until hover unless selected.
  • Add a trash icon for single-date deletion.
  • Use stopPropagation() so controls do not expand or collapse the date group.

Step 3: Implement Date Selection Logic

  • Add toggleDateSelection(dateMeetingIds, event).
  • If every meeting in the date is selected, unselect them.
  • Otherwise add every meeting ID from that date to selectedMeetingIds.
  • Keep Escape-to-clear behavior unchanged.

Step 4: Support Single Date Group Delete

  • Add state like dateDeleteModalState.
  • Open confirmation text like Delete all N meetings from DATE?.
  • On confirm, delete all meeting IDs in that date group.
  • Navigate home if the active meeting was deleted.

Step 5: Reuse Bulk Delete

  • Keep the existing bottom selection bar.
  • Let selected date groups populate selectedMeetingIds.
  • Keep the label as N meetings selected to handle mixed date and row selections clearly.

Step 6: Add Meeting Row Controls Inside By Date

  • Reuse DraggableMeetingRow or create a non-draggable shared row if needed.
  • Pass selection, edit, delete, and navigation handlers.
  • Preserve click-to-open behavior.

Step 7: Extract Shared Delete Helper

  • Move the bulk delete loop into a helper like deleteMeetingIds(ids).
  • Use it from bulk delete and single date group delete.
  • Keep single meeting delete as-is unless routing through the helper stays clean.
